# OpenClaw 使用场景——知识沉淀

## 核心观点

Knowledge 模块是"外部知识沉淀层"，负责收录文章、网页、笔记、视频转写等外部材料，用_index.md 做统一的摘要检索入口，让系统具备稳定的知识进入、整理、检索和复用机制。

## 主要内容

### Knowledge 模块是什么

knowledge 模块本质上是一个"外部知识沉淀层"。它通常负责这几件事：

1. **收录**文章、网页、笔记、视频转写等外部材料
2. **检索**用 `_index.md` 做统一的摘要检索入口
3. **存储**用正式条目保存"摘要 + 全文"
4. **流转**用 `inbox/` 管理待处理、已处理、待复核的素材流转

它解决的不是"多建一个文件夹"这么简单，而是让系统具备一套稳定的知识进入、整理、检索和复用机制。

### 你应该怎么用

最简单的用法：

1. 打开你常用的 AI 对话工具
2. 告诉 AI：请基于当前项目来处理，不需要我额外提供项目路径
3. 再给它一个公开链接，让它自己去访问那份引导提示词
4. 明确告诉它：按链接里的提示词执行

**推荐话术**：
```
我想在我当前的项目/知识库体系里接入一个 knowledge 模块。
请不要重构整套系统，而是先理解我现在的结构，再增量接入。
请直接基于当前项目进行扫描和处理，不需要我再提供项目路径。
下面这个链接里是给你执行用的引导提示词，请先访问并阅读，再按里面的要求执行：
https://ai.weijennie.com/assets/knowledge-guide.md
```

### 关于 skill 的部分

**Skill 名称**: `knowledge-kb`

**作用**：
- 把 pending 文件夹中的文件夹归档索引到知识库中
- 把音视频内容处理为知识库文档

**安装方式**：
- 手动下载：knowledge-kb-main.zip (15.82KB)
- GitHub: https://github.com/dingshuxin353/knowledge-kb

**处理音视频需要**：
- 阿里百炼的 API key
- 配置到 config 文件中
